# VRITSC Design System

Sistema de diseño visual para la **Vicerrectoría de Investigación, Desarrollo, Innovación y Emprendimiento** del **Instituto Técnico Superior Comunitario (ITSC)**.

> **Estado:** documento técnico base para validación institucional.  
> **Uso recomendado:** repositorio `vritscdesign`, diseño gráfico, UI/UX, frontend, presentaciones, documentos, dashboards, plantillas institucionales y generación asistida por IA.  
> **Archivo:** `README.md`

---

## 1. Descripción general

Este README define la base técnica, visual y operativa del sistema de diseño institucional para la Vicerrectoría de Investigación, Desarrollo, Innovación y Emprendimiento del ITSC.

El sistema se construye a partir de:

- La identidad madre del **Instituto Técnico Superior Comunitario (ITSC)**.
- Los logotipos institucionales entregados en formato SVG.
- La imagen del campus institucional.
- Las presentaciones y documentos de la Vicerrectoría I+D+i / IDI.
- Las plantillas documentales institucionales.
- La página web oficial `www.itsc.edu.do`.
- Los informes visuales semanales y documentos de gestión compartidos.

El objetivo es establecer una guía clara para producir piezas visuales coherentes, profesionales y escalables sin alterar la identidad oficial del ITSC.

---

## 2. Propósito del sistema

El sistema debe permitir que diseñadores, desarrolladores, equipos institucionales e inteligencias artificiales generativas puedan crear materiales visuales consistentes para:

- Documentos institucionales.
- Presentaciones ejecutivas.
- Informes semanales o de gestión.
- Dashboards de seguimiento.
- Formularios y fichas de proyecto.
- Plantillas Word, PDF y PowerPoint.
- Interfaces web o sistemas internos.
- Piezas para comunicación institucional.
- Recursos visuales de innovación, investigación, datos, emprendimiento y desarrollo tecnológico.

## 4. Diagnóstico visual

### 4.1 Identidad madre ITSC

La identidad madre del ITSC comunica:

- Educación superior técnica.
- Institución pública.
- Servicio comunitario.
- Formación profesional.
- Modernización académica.
- Vinculación con sectores productivos.
- Transparencia y rendición de cuentas.

El sistema visual de la Vicerrectoría debe respetar esta base y extenderla hacia un lenguaje más técnico, innovador y orientado a datos.

### 4.2 Subidentidad Vicerrectoría IDI / I+D+i+E

La Vicerrectoría requiere un sistema visual con mayor énfasis en:

- Investigación aplicada.
- Desarrollo tecnológico.
- Innovación institucional.
- Emprendimiento.
- Inteligencia de datos.
- Microcredenciales.
- Transformación digital.
- Prototipado.
- Cooperación interinstitucional.
- Evidencia, métricas y visualización de datos.

### 4.3 Hallazgos visuales principales

| Hallazgo | Implicación para el sistema |
|---|---|
| La marca madre usa azul/gris/rojo institucional. | Mantenerla como capa oficial y formal. |
| Los informes recientes usan dark UI, cian, menta y violeta. | Usar como lenguaje secundario para innovación, datos y presentaciones ejecutivas. |
| Las plantillas de proyecto usan tablas azules y estructuras densas. | Normalizar estilos de formularios, espaciado y jerarquía. |
| La web institucional tiene componentes de servicios, noticias, plataformas y accesibilidad. | Tomar como referencia para navegación, cards y contenidos públicos. |
| Existen estilos visuales distintos entre documentos, informes y presentaciones. | Crear tokens, reglas y componentes para evitar dispersión visual. |

## 5. Principios de diseño

### 5.1 Principios rectores

1. **Institucionalidad clara**  
   Toda pieza debe sentirse oficial, confiable y alineada al ITSC.

2. **Innovación aplicada**  
   Las piezas de la Vicerrectoría pueden usar recursos visuales modernos, tecnológicos y orientados a datos, sin romper la identidad madre.

3. **Jerarquía ejecutiva**  
   Los documentos deben facilitar lectura rápida: título, contexto, dato clave, evidencia y decisión.

4. **Claridad sobre decoración**  
   Ningún recurso visual debe reducir la comprensión del contenido.

5. **Escalabilidad**  
   El sistema debe funcionar en Word, PDF, PowerPoint, web, dashboards y documentos técnicos.

6. **Accesibilidad**  
   El diseño debe considerar contraste, legibilidad, navegación y claridad para distintos usuarios.

7. **Consistencia documental**  
   Todos los informes, plantillas y piezas deben compartir estructura, nombres, colores, tipografía y reglas de composición.

## 7. Uso del logo

### 7.1 Versiones aceptadas

| Versión | Archivo | Uso recomendado |
|---|---|---|
| Logo oficial a color | `Logo ITSC.svg` | Fondos claros, documentos oficiales, encabezados formales. |
| Logo blanco | `Logo ITSC - Blanco.svg` | Fondos oscuros, presentaciones dark, portadas ejecutivas. |
| Logo gris | `Logo ITSC - Gris.svg` | Documentos sobrios, marcas de agua, fondos claros de baja prioridad. |

### 7.2 Reglas obligatorias

- No alterar proporciones del logo.
- No redibujar el símbolo.
- No cambiar colores del logo oficial.
- No aplicar sombras agresivas al logo.
- No colocar sobre fondos de bajo contraste.
- No deformar horizontal o verticalmente.
- No usar efectos 3D, biselados o brillos no institucionales.
- No usar el logo como decoración repetitiva sin intención.

### 7.3 Área de seguridad

Usar como mínimo una distancia equivalente a la altura de las letras `ITSC` alrededor del logo.

```text
[ espacio mínimo ]  LOGO ITSC  [ espacio mínimo ]
```

### 7.4 Tamaño mínimo recomendado

| Medio | Tamaño mínimo |
|---|---:|
| Documento impreso | 20 mm de ancho |
| Presentación 16:9 | 28 px de alto mínimo |
| Web desktop | 36 px de alto mínimo |
| Mobile | 28 px de alto mínimo |
| Marca de agua | 12 % a 18 % del ancho del lienzo, con baja opacidad |

### 7.5 Marca de agua institucional

Para presentaciones dark se permite usar el sello institucional en gran formato, con opacidad reducida:

```css
opacity: 0.04 - 0.10;
blend-mode: screen;
```

No debe competir con el contenido ni reducir la legibilidad.

## 9. Tipografía

> **Pendiente de validación institucional:** no se confirmó una familia tipográfica oficial del ITSC en los archivos entregados. El sistema recomienda una jerarquía sans serif moderna, técnica y legible.

### 9.1 Familias recomendadas

| Uso | Familia recomendada | Alternativas |
|---|---|---|
| UI / dashboard / web | `Inter` | `Roboto`, `Source Sans 3`, `Arial` |
| Presentaciones ejecutivas | `Inter Tight` | `Montserrat`, `Sora`, `Aptos Display` |
| Documentos Word/PDF | `Aptos` | `Calibri`, `Arial`, `Source Sans 3` |
| Datos / métricas / folios | `IBM Plex Mono` | `Roboto Mono`, `JetBrains Mono` |

### 9.2 Jerarquía tipográfica

| Token | Tamaño | Peso | Uso |
|---|---:|---:|---|
| `display-xl` | 72-96 px | 700-800 | Portadas, números de sección, impacto visual. |
| `display-lg` | 48-64 px | 700 | Títulos de presentación. |
| `heading-1` | 36-48 px | 700 | Títulos principales. |
| `heading-2` | 28-32 px | 650 | Secciones. |
| `heading-3` | 20-24 px | 600 | Subtítulos. |
| `body-lg` | 18-20 px | 400 | Texto destacado. |
| `body-md` | 15-16 px | 400 | Texto base UI/web. |
| `body-sm` | 13-14 px | 400 | Notas, captions, ayudas. |
| `label` | 11-12 px | 600 | Etiquetas, folios, headers técnicos. |
| `mono-stat` | 32-72 px | 700 | Métricas, indicadores y data cards. |

### 9.3 Reglas tipográficas

- Usar títulos grandes y claros en presentaciones.
- Mantener alto contraste entre títulos y fondo.
- Evitar más de dos familias por pieza.
- Usar monoespaciada solo para folios, números técnicos, etiquetas o datos.
- Evitar textos largos en mayúsculas sostenidas, salvo labels cortos.
- Mantener interlineado generoso en documentos: `1.25 - 1.5`.
- En presentaciones, usar poco texto y priorizar datos clave.

## 16. Presentaciones institucionales

### 16.1 Formato recomendado

- Relación: 16:9.
- Tamaño: 1920 x 1080 px.
- Margen seguro: 88 px horizontal, 72 px vertical.
- Logo: esquina superior izquierda o marca de agua derecha.
- Folio: esquina superior derecha.
- Pie: `VICERRECTORÍA IDI` o unidad correspondiente.

### 16.2 Tipos de slides

| Tipo | Uso |
|---|---|
| Cover | Apertura de presentación. |
| Agenda | Vista general de temas. |
| Section divider | Cambio de bloque temático. |
| Metric slide | Indicadores y resultados. |
| Process slide | Flujo de procesos o metodología. |
| Project slide | Estado de proyecto estratégico. |
| Timeline | Cronograma. |
| Closing | Cierre, síntesis y próximos pasos. |

### 16.3 Estilo dark ejecutivo

Recomendado para informes de Vicerrectoría IDI.

```css
background: #0B1224;
color: #FFFFFF;
accent: #22D7E7;
secondary-accent: #51E6CF;
```

Patrones visuales:

- Retícula técnica de fondo al 3 % - 8 % de opacidad.
- Glow cian suave en zonas de énfasis.
- Cards translúcidas.
- Métricas grandes.
- Titulares con palabras clave en gradiente cian/menta.

### 16.4 Reglas para slides

- No usar más de una idea principal por slide.
- Priorizar datos grandes sobre párrafos largos.
- Usar máximo 3 acentos cromáticos por slide.
- Mantener consistencia en folios, títulos y pies.
- No saturar con logos.
- No usar fondos fotográficos si comprometen la lectura.

## 18. Accesibilidad

### 18.1 Contraste

- Texto normal: mínimo WCAG AA `4.5:1`.
- Texto grande: mínimo `3:1`.
- Botones: contraste claro entre fondo y texto.
- No depender solo del color para comunicar estado.

### 18.2 Reglas de legibilidad

- No usar texto menor a 12 px en UI.
- No usar texto menor a 9 pt en documentos impresos.
- Evitar mayúsculas extendidas en párrafos.
- Mantener líneas de texto entre 45 y 75 caracteres.
- Usar suficiente interlineado.

### 18.3 Accesibilidad en dashboards

- Incluir labels en gráficos.
- Usar patrones o iconos además de color.
- Permitir lectura de datos en tablas.
- Evitar animaciones parpadeantes.
- Mantener foco visible en navegación por teclado.

## 23. Convención de nombres de archivos

### 23.1 Regla general

Usar nombres claros, sin acentos, sin caracteres especiales y con fecha cuando aplique.

```text
area_tipo_documento_tema_version_fecha.ext
```

### 23.2 Ejemplos

```text
idi_informe_semanal_mayo_2026_v01.pdf
idi_presentacion_microcredenciales_v02.pptx
itsc_logo_color.svg
idi_tokens_v01.json
idi_dashboard_kpis_v01.fig
idi_ficha_proyecto_innovacion_v03.docx
```

### 23.3 Versionado

```text
v01 = primera versión
v02 = segunda revisión
v03 = revisión validada
final = solo cuando esté aprobado institucionalmente
```

Evitar:

```text
final_final.pdf
nuevo 2.pptx
copia definitiva.docx
```

- Evitar subir versiones sensibles sin revisión.
